## ✅ 검증 계획

### 법령별 검증 (각 법령 수집 후)

```bash
# 품질 검증
python scripts/legal/validate_neo4j_quality.py

# 확인 사항:
# - 조문 개수
# - 참조 개수
# - 데이터 무결성
```

### 최종 검증 (전체 완료 후)

**검증 항목**:

1. **노드 증가 확인**
   ```cypher
   MATCH (l:Law)
   RETURN count(l) AS law_count
   // Before: 35개 → After: 46개 (예상)
   ```

2. **타법 참조 증가 확인**
   ```cypher
   MATCH ()-[r:CROSS_LAW_REFERS_TO]->()
   RETURN count(r) AS cross_law_refs
   // Before: 859개 → After: 1,227개 (예상 +368)
   ```

3. **법령별 타법 참조 통계**
   ```cypher
   MATCH (s:Article)-[r:CROSS_LAW_REFERS_TO]->(t:Article)
   WHERE s.law_name <> t.law_name
   RETURN t.law_name AS target_law, count(r) AS ref_count
   ORDER BY ref_count DESC
   LIMIT 20
   ```

4. **데이터 무결성**
   - 고아 노드: 0개
   - 깨진 참조: 0개
   - 중복 ID: 0개

## 💡 최적화 팁

### 1. 병렬 처리

**법령별 독립 작업**:
```bash
# 법령 3개를 병렬로 수집 (시간 절약)
python likms/scrapers/law_scraper.py --law-name "전자정부법" &
python likms/scrapers/law_scraper.py --law-name "개인정보 보호법" &
python likms/scrapers/law_scraper.py --law-name "간호법" &
wait
```

### 2. 자동화 스크립트

```bash
# 전체 자동화 스크립트 작성
#!/bin/bash
LAWS=(
    "전자정부법"
    "개인정보 보호법"
    "간호법"
    # ... 나머지 8개
)

for law in "${LAWS[@]}"; do
    echo "Processing: $law"
    python likms/scrapers/law_scraper.py --law-name "$law"
    python scripts/legal/parse_articles.py
    python scripts/legal/extract_references.py
done

# Neo4j 통합 (일괄)
python neo4j/scripts/integrate_legal_to_neo4j.py --clear-db
python scripts/legal/integrate_cross_law_to_neo4j.py
```
